## Decisions
1. **Release build over debug for testing**
   - **Choice**: Use `assembleRelease` instead of `assembleDebug`
   - **Rationale**: Debug builds open Expo Dev Launcher instead of the app; release embeds JS bundle automatically
   - **Alternative rejected**: Manual `expo export` + copy HBC to assets (works but fragile)

2. **VPS-only Android builds**
   - **Choice**: No local Android SDK, all builds via SSH to `automation@69.62.108.82`
   - **Rationale**: VPS has dedicated resources, no Windows path issues, 26GB disk savings locally
   - **Alternative rejected**: Local builds (slow, path issues, disk waste)

3. **Mobile MCP via MetaMCP for device interaction**
   - **Choice**: Use `http://127.0.0.1:8750/servers/mobile/mcp` Streamable HTTP protocol
   - **Rationale**: Provides structured device control (screenshot, tap, UI tree, shell) without raw ADB
   - **Alternative rejected**: Direct ADB commands (less structured, no UI tree)

4. **Manual VPS sync instead of cherry-pick**
   - **Choice**: Use `sed` and manual edits on VPS
   - **Rationale**: Fork flat structure (`happy/`) vs upstream monorepo (`packages/happy-app/`) makes cherry-pick impossible
   - **Alternative rejected**: Git cherry-pick (path mismatch breaks all diffs)

## Problems
1. **Expo Dev Launcher blocks app**
   - **Symptom**: Debug APK opens `DevLauncherActivity` with "Connect to server" screen
   - **Cause**: Debug builds don't embed JS bundle by default
   - **Solution**: Either export+copy HBC manually, or use release build
   - **Prevention**: Always use `assembleRelease` for testing

2. **Phone lock screen blocking automation**
   - **Symptom**: `mCurrentFocus=Bouncer` — PIN required
   - **Cause**: Samsung device has PIN lock, `locksettings clear` requires current PIN
   - **Solution**: User physically unlocked the phone
   - **Prevention**: Disable lock screen before automation sessions

3. **ADB disconnection mid-session**
   - **Symptom**: `adb devices` empty after daemon restart
   - **Cause**: ADB daemon restarted, USB authorization lost
   - **Solution**: Reconnect USB cable
   - **Prevention**: Keep `svc power stayon usb` active

4. **VPS build timeouts**
   - **Symptom**: Gradle builds exceed 600s SSH timeout
   - **Cause**: First clean build on VPS with limited RAM (10GB used)
   - **Solution**: Poll `pgrep -f gradle` in loop, incremental builds much faster
   - **Prevention**: Use polling pattern for long builds
